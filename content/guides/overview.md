本文档系统梳理了目前大模型开发领域的三大核心方向，详细解释了各个方向的专有名词，并附带了权威的学习资源链接与面试核心词汇，帮助开发者建立清晰的技术认知地图。

## 核心方向概览

大模型开发岗位从上层应用到底层算法，可以切分为三大核心方向。

### 方向一：大模型应用层开发 (AI Application / Agent Engineer)
> **定位**：如同"汽车改装厂与内饰设计师"。拿到现成的模型 API，通过工程化手段解决特定业务场景的问题。门槛相对适中，是目前需求量最大、最适合广大开发者（前端、后端、全栈）切入的方向。

#### 1. Prompt Engineering (提示词工程)
*   **概念**：不需要高级框架，通过精心设计系统提示词（System Prompt）、提供示例，让大模型稳定输出指定格式（如 JSON）的结果。
*   **面试常问词汇/专有名词**：
    *   **Few-Shot (少量示例提示)**：在提问前先给模型几个正确回答的例子。
    *   **CoT (Chain of Thought 链式思考)**：让模型"一步步思考（Let's think step by step）"，从而提升逻辑推理能力。
    *   **结构化输出 / Pydantic**：确保大模型输出的 JSON 格式完全符合后端代码期望的数据结构。

#### 2. RAG (检索增强生成)
*   **概念**：为大模型外接知识库（相当于给大模型发"开卷考试参考书"），解决模型"胡说八道（幻觉）"和"不知道企业内部私密数据"的问题。
*   **面试常问词汇/专有名词**：
    *   **Chunking (分块)**：将长文档切分成小块，不仅按字数切，还需考虑按语义（段落、标题）切分。
    *   **Vector DB (向量数据库)**：用来存储文档特征向量的数据库，如 Milvus, Qdrant。
    *   **Embedding (词嵌入)**：把人类文字翻译成机器能懂的多维数字向量。
    *   **Hybrid Search (混合检索)**：结合 **关键词匹配 (BM25)** 和 **向量相似度匹配**，提升检索准确率。
    *   **Reranker (重排序模型)**：对初步检索出的文档进行二次精准打分。
*   **推荐学习资源**：
    *   [LlamaIndex 高级 RAG 官方教程](https://docs.llamaindex.ai/en/stable/optimizing/advanced_retrieval/advanced_retrieval/)：学习 Reranker 和混合检索的最佳官方文档。
    *   [Ragas 官方文档 (RAG 评估)](https://docs.ragas.io/en/latest/)：量化评估你的 RAG 系统（如算出 Top-3 命中率）。

#### 3. Workflow 与 Agentic (工作流与智能体)
*   **概念**：让大模型不仅能"说话"，还能"做事"。
    *   **Workflow（工作流）**：像"流水线工人"，强调**可控性**。开发者规定好 SOP，大模型只在固定节点上完成特定任务。
    *   **Agentic（自主智能体）**：像"聪明的主管"，强调**自主性**。只给总目标，模型自己决定规划路径、选择工具并执行。这是目前应用层**含金量最高**的技术栈。
*   **面试常问词汇/专有名词**：
    *   **ReAct (Reason + Act)**：智能体一边推理思考，一边采取行动的经典框架。
    *   **Tool Use / Function Calling (工具调用)**：大模型向外部世界发送请求的能力。
    *   **Human-in-the-loop (人类在环)**：在智能体执行关键高风险任务时，暂停并要求人类确认。
*   **推荐学习资源**：
    *   [吴恩达 Agentic Design Patterns 免费课程 (DeepLearning.AI)](https://www.deeplearning.ai/the-batch/how-agents-can-improve-llm-performance/)：强烈推荐，讲解了反思、工具使用、规划、多智能体协作四种核心设计模式。
    *   [LangGraph 官方文档](https://langchain-ai.github.io/langgraph/)：构建复杂、可控多智能体应用的主流框架。

### 方向二：大模型工程化与平台层 (LLMOps / Infra Engineer)
> **定位**：如同"修车厂长与底盘调校师"。负责大模型的高并发部署、推理加速以及自动化监控平台的搭建，需要极强的底层后端工程能力。

#### 1. 模型推理加速与部署 (Serving)
*   **概念**：研究如何把开源模型（如 Qwen, Llama）高效部署在企业服务器 GPU 上，封装成支持高并发请求的 API 服务。
*   **面试常问词汇/专有名词**：
    *   **vLLM**：目前工业界最流行的开源推理加速框架。
    *   **KV Cache (键值缓存)**：大模型生成文本时，缓存之前计算过的特征，极大地加快后续生成速度。
    *   **PageAttention**：vLLM 的核心技术，借鉴操作系统的虚拟内存分页管理，解决 GPU 显存碎片化问题。
    *   **TTFT (Time To First Token)**：首字响应时间，衡量大模型服务体验的最核心指标。
    *   **Streaming (流式传输/SSE)**：让大模型像打字机一样一个字一个字吐出结果的技术。
*   **推荐学习资源**：
    *   [vLLM 官方文档](https://docs.vllm.ai/en/latest/)：了解大模型底层加速原理必读。

#### 2. 评测监控与低代码平台 (Evaluation & LLMOps)
*   **概念**：建设自动化评测系统，监控模型运行成本；或搭建企业级可视化拖拽开发平台（让不懂代码的人也能开发 AI）。
*   **推荐学习资源**：
    *   [Dify.ai 官方文档](https://docs.dify.ai/)：极佳的开源 LLMOps 平台，适合研究源码架构。
    *   [LangSmith](https://smith.langchain.com/)：学习如何做大模型调用的 Trace（链路追踪）和 Token 成本监控。

### 方向三：大模型算法与微调层 (Model Tuning / Algorithm Engineer)
> **定位**：如同"发动机研发工程师"。直接修改模型内部的权重参数（给大模型做「脑部手术」），门槛最高，通常要求人工智能/深度学习相关专业的硕士/博士背景。

#### 1. 监督微调 (SFT - Supervised Fine-Tuning)
*   **概念**：给大模型投喂成千上万条特定格式的"问答对"数据，改变其内部神经元权重，把它变成特定领域的专家或改变其说话语气。
*   **面试常问词汇/专有名词**：
    *   **Instruction Tuning (指令微调)**：SFT 的一种，教会模型按照人类的指令行事。
    *   **PEFT (参数高效微调)**：不用更新模型全部几百亿个参数，只更新极小一部分，从而大幅节省显存的方法。
    *   **LoRA / QLoRA**：目前最主流的 PEFT 算法，允许开发者在单张消费级显卡（如 4090）上微调 7B 级别的大模型。
*   **推荐学习资源**：
    *   [LLaMA-Factory Github 仓库](https://github.com/hiyouga/LLaMA-Factory)：国内火爆的微调工具箱（支持 Web 图形界面），极大降低微调门槛。

#### 2. 预训练与强化学习对齐 (Pre-training / RLHF)
*   **概念**：
    *   **Pre-training (预训练)**：从零开始，给模型喂几万亿字的互联网原始海量语料，让它学会"人类的语言规律"。
    *   **RLHF (基于人类反馈的强化学习)**：原始模型可能会输出有害内容，RLHF 通过让人类给模型的回答打分，教导模型变得"安全、有益、符合人类价值观"。
*   **备注**：此领域属于大厂核心基座团队专属，**面试时仅需了解名词概念，切忌宣称自己精通**。

## 💡 面试求职与进阶路线建议

1. **专注破局**：如果你主攻应用开发，请死磕 **方向一（高级 RAG + Agentic）**。能把这两个方向的词汇（如 Hybrid Search, ReAct, TTFT）结合具体项目说透，已经能打败大多数普通求职者。
2. **拓展工程视野**：尝试跑通一次 **LLaMA-Factory 的微调（SFT）** 以及 **使用 vLLM 本地部署**。这能让你的简历脱颖而出，证明你具备「全链路」开发思维。
3. **面试话术模板**："我目前的核心聚焦在应用层，特别是利用混合检索解决 RAG 幻觉，以及使用 LangGraph 搭建智能体工作流。同时为了让应用更好落地，我了解 vLLM 的推理加速原理，也清楚当 Prompt 无法解决特定领域知识欠缺时，可以通过 LoRA 进行 SFT 微调。"
